# @MistyBridge — Resume Craft
# 阶段 5 — 篇幅自检与模块调整

> 当前阶段：5 | 上一阶段：4 | 下一阶段：6 | 可跳过：否

## 本阶段目标

将阶段4生成的内容估算为实际篇幅，与阶段2设定的篇幅目标对比。
如偏差过大，**动态分析**当前模块构成和用户实际情况，给出调整建议。
用户选择后执行调整并重新估算，直到达标。完成后进入阶段6。

## 前置条件

- 阶段4内容填充已完成
- 篇幅目标来自阶段2（简略/正常/详细）
- 模板Section顺序来自阶段3
- 岗位信息（如有）来自阶段1
- 用户档案来自阶段2

## 篇幅估算标准

| 篇幅目标 | 汉字数 | A4页数 |
|---------|--------|--------|
| 简略 | 800-1500字 | 1-2页 |
| 正常 | 1500-3000字 | 2-4页 |
| 详细 | ≥ 3000字 | 4页以上 |

三档均有上下界：低于下限需详写补充，超出上限需简写压缩。

估算方法：汇总所有 Section 中已润色内容的汉字总数（不含姓名/联系方式/技能标签）。

如果阶段2选择了「详细」且当前已达3000字 → 仅确认达标，不强制调整。

---

## 核心原则：动态分析，不套模板

**禁止**使用固定5个选项的模板。每次必须根据以下因素动态生成调整建议：

1. **实际模块构成**：阶段3锁定的 Section 列表，有几个列几个
2. **字数占比**：各模块当前汉字数及占比
3. **岗位相关性**（如有JD）：模块内容命中JD关键词的数量，决定该模块的重要性权重
4. **数据丰富度**：用户档案中该模块的原始数据量，决定详写空间
5. **学科模板权重**：当前学科对该模块的 emphasis（dominant > high > normal），dominant 模块不应被建议删除

## 执行步骤

### 5.0 动态分析当前篇幅

遍历阶段3锁定的 Section 列表，统计字数，生成表格：

```
📏 篇幅自检

目标篇幅：<简略(800-1500字) / 正常(1500-3000字) / 详细(≥3000字)>
当前合计：<N> 字 — <超标X字 / 不足X字 / 达标>

各模块统计：
┌────────────────┬──────────┬──────────┬────────────┐
│ 模块           │ 汉字数    │ 占比     │ JD命中      │
├────────────────┼──────────┼──────────┼────────────┤
│ <模块1>        │   XXX    │  XX%     │ XX个关键词  │
│ <模块2>        │   XXX    │  XX%     │ XX个关键词  │
│ ...（按阶段3 Section顺序，有几个列几个）          │
├────────────────┼──────────┼──────────┼────────────┤
│ 合计           │  XXXX    │ 100%     │            │
└────────────────┴──────────┴──────────┴────────────┘
```

### 5.1 篇幅判定与动态建议

#### 情况 A — 篇幅超标（当前字数 > 目标上限）

根据动态分析**智能排序**压缩/删除建议：

**排序原则**：
- 占比高 + JD命中少 → 优先建议简写
- 占比低 + JD命中少 + 非核心模块 → 优先建议删除
- 占比虽高但JD命中多 → 建议轻度压缩，标注「核心模块」
- 学科 emphasis=dominant → 不列入删除建议

```
⚠️ 篇幅超标：当前 X 字，目标 X-X 字，超出 X 字。

── 建议简写（按优先级排序）──
  [1] <模块A>（X字，占X%） — <动态理由> → 预计可压缩至X字
  [2] <模块B>（X字，占X%） — <动态理由> → 预计可压缩至X字
  ...

── 建议删除（影响较小）──
  [D] 删除模块 — 展开选择列表

输入编号（可多选），[自动优化]，或 [取消]。
```

选择 [D] 后**动态排序删除建议**：

```
🗑 可删除模块（按影响从小到大）：

  [1] <模块X>（X字） — JD未命中，学科权重低，删除影响最小
  [2] <模块Y>（X字） — JD命中1项但可合并到其他模块
  ...

⚠️ 不建议删除：<模块Z>（JD命中X项 / 学科核心模块）

输入编号，或 [取消]。
```

**压缩规则**：
- 简写时保持阶段4选定的模块风格方向不变
- 不丢失JD命中关键词
- 优先合并同类项、删除修饰词

#### 情况 B — 篇幅不足（当前字数 < 目标下限）

根据动态分析**智能排序**详写/新增建议：

**排序原则**：
- 数据丰富 + JD命中多 → 优先建议详写（有料可写，写了有用）
- 数据丰富 + JD命中少 → 次要建议详写
- 数据不足 → 标注「档案数据有限，详写空间小」
- 有未使用的档案数据 → 优先推荐新增对应模块

```
⚠️ 篇幅不足：当前 X 字，目标 X-X 字，不足 X 字。

── 建议详写（按扩展潜力排序）──
  [1] <模块A>（X字） — 档案中还有X条未使用经历可展开 → 预计可达X字
  [2] <模块B>（X字） — 当前内容较简，可补充技术细节 → 预计可达X字
  ...

── 建议新增（按数据就绪度排序）──
  [N] 增加新模块 — 展开推荐列表

输入编号（可多选），[自动优化]，或 [取消]。
```

选择 [N] 后**动态生成推荐模块列表**：

```
📦 推荐新增模块

根据档案未使用数据和当前模板空缺，推荐以下模块：

  [A] <推荐模块1> — 档案已有：<具体数据摘要>
  [B] <推荐模块2> — 学科模板建议添加，需补充内容
  [C] 自定义模块 — 输入模块名称和内容方向

推荐依据：
  · 阶段3学科模板中未使用的 Section：<列表>
  · 档案中阶段4未覆盖的数据：<列表>

输入编号，或 [取消]。
```

**推荐生成规则**（动态，不硬编码）：
1. 扫描 `data/profiles/<档案名>.json`，找出阶段4未使用的字段（certifications / patents / publications / interests / 未被引用的 projects 等）
2. 交叉比对阶段3学科模板的完整 sections 列表，找出当前未使用的
3. 有档案数据的 → 排前面，标注具体内容摘要
4. 仅模板建议的 → 排后面，标注「需补充」
5. 最后一个选项永远是「自定义模块」

选定新模块后进入**新增模块填充**子流程：

```
📝 新增模块填充 — <模块名>

[自动提取] — 从档案中提取并润色
[手动录入] — 粘贴或口述
[跳过]
```

填充后重新估算。

**扩展规则**：
- 仅从用户档案已有数据中提取，不编造
- 详写时保持模块风格方向不变
- 每次调整后重新估算

#### 情况 C — 篇幅达标

```
✅ 篇幅达标：当前 X 字，目标 X-X 字。

进入阶段6 — 渲染导出。
```

### 5.2 执行调整并重新估算

每次调整后立即重新统计字数：

```
🔧 调整结果：

  ✓ <模块A> — 简写：520字 → 320字（-200字）
  ✓ <模块B> — 删除（-280字）

📏 调整后：合计 X 字 — <仍超标/仍不足/已达标>

[继续调整]  [接受现状（偏差<10%）]  [恢复]
```

若偏差 < 10% 且用户接受，视为达标。

### 5.3 完成

```
✅ 阶段5完成

📏 篇幅确认：
  目标：<简略/正常/详细>（X-X字）
  实际：X 字 ✅ 达标
  调整记录：<记录每步操作>

进入阶段6 — 渲染导出。
```
