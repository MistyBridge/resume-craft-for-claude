# @MistyBridge — Resume Craft
# 阶段 4 — 逐模块内容填充 + 岗位匹配

> 当前阶段：4 | 上一阶段：3 | 下一阶段：5 | 可跳过：否

## 本阶段目标

按模板Section顺序，**每个模块整体选择一种风格方向**，统一生成该模块下所有内容。
篇幅设置（来自阶段2）决定文本的精简/展开程度。
完成后生成匹配报告，进入阶段5（篇幅自检与模块调整）。

## 前置条件

- 模板已在阶段3锁定（Section 顺序已确定）
- 档案数据来自阶段2
- 篇幅设置来自阶段2（简略/正常/详细）
- 岗位信息（如有）来自阶段1

## 核心红线

- **只做表达增强，不编造内容**
- **量化数据仅在用户明确提供时使用**
- **不确定的改写方向 → 提供选项让用户决策**
- **绝不编造任何数字**

## 篇幅控制策略

| 篇幅 | 每条经历目标字数 | 经历保留规则 | 项目展示 |
|------|-----------------|-------------|---------|
| 简略（800-1500字） | 40-60字 / 条 | 仅保留核心3-4条，其余合并或删除 | 仅保留1-2代表作 |
| 正常（1500-3000字） | 60-100字 / 条 | 核心经历展开，次要1-2句带过 | 展示主要项目 |
| 详细（≥3000字） | 100-150字 / 条 | 全部保留并充分展开 | 完整展示所有项目 |

```
岗位信息：<有/无>

  <Section顺序列表>

如有需要也可针对单条内容微调。
```

#### 如果阶段1有岗位信息，创建输出目录

```bash
mkdir -p "data/output/<公司名>/<公司名-岗位名>"
```

### 4.1 内部润色策略（3-Pass，对用户透明）

对每个模块统一应用：

**Pass 1 — 篇幅适配**
- 根据篇幅目标调整每条内容的字数和信息密度
- 简略（800-1500字）：压缩冗余、合并同类项、删除修饰词
- 正常（1500-3000字）：标准 STAR 结构，核心经历充分展开
- 详细（≥3000字）：完整保留，适当补充上下文和技术细节

**Pass 2 — 风格统一**
- 模糊表述 → 专业术语（按风格方向调整术语深度）
- 统一中英文术语
- 确保模块内所有条目风格一致
- 如有JD：自然融入JD关键词

**Pass 3 — 一致性检查**
- 时态统一、术语统一、日期格式统一
- 模块内条目详略程度均匀

### 4.2 按 Section 顺序逐模块处理

按阶段3锁定的 Section 顺序，**每次处理一个 Section（模块）**。

展示模块概览 + 风格选择：

```
📝 <模块名> — 选择整体风格（<当前>/<总数>）

该模块共 <N> 条内容：
  1. 负责微信小程序性能优化
  2. 搭建前端工程化体系
  3. 开发公共组件库

请选择统一风格：

[A] 技术深度侧重
    强调技术架构、实现细节、工具链和工程方法，
    适合研发/技术岗位简历

[B] 业务成果侧重
    强调业务影响、用户价值和量化成果，
    适合偏业务/管理岗位简历

[C] 简洁专业侧重
    精炼表达，一句话讲清核心贡献，
    适合篇幅受限或注重可读性的简历

[D] 保持原样
    保留原始表述，仅做格式和术语统一

当前篇幅：<简略/正常/详细>

请选择：A / B / C / D
或输入「逐条定制」进入逐条选择模式
```

**风格方向差异示例**：

```
原始：「负责微信小程序性能优化」

[A] 技术深度 → 针对小程序渲染链路进行系统分析，通过分包加载与预请求
              策略优化首屏加载性能，重构核心模块以降低运行时内存占用。

[B] 业务成果 → 主导小程序性能优化，首屏加载时间从3.2s降至1.1s（↓66%），
              日活用户页面流失率降低40%，支撑核心业务稳定增长。

[C] 简洁专业 → 主导微信小程序端到端性能优化，显著提升加载速度与用户体验。

[D] 保持原样 → 负责微信小程序性能优化
```

**重要**：同模块下所有条目自动应用同一风格，用户无需逐条选择。
风格差异必须明显（技术深度/业务成果/简洁专业），让选择有意义。

### 4.3 模块级批量操作

用户对当前模块可用以下操作：
- `A / B / C / D` → 该模块全部条目应用此风格
- `逐条定制` → 进入逐条选择模式，恢复为逐条 A/B/C/D
- `换一批` → 为当前模块重新生成风格预览（换示例条目展示）
- `跳过此模块` → 该模块所有条目保持原样

### 4.4 对无法量化的内容

模糊量化词（翻倍、大幅、明显、显著等）保留原词，在旁边标记：

```
经过优化后，系统性能得到大幅提升 [建议补充：提升了多少？响应时间从X到Y？]
```

**绝不编造任何数字。**

每完成一个模块，输出简短确认：

```

[C] 简洁专业侧重  [D] 保持原样  [E] 恢复模块风格

请选择：
```
